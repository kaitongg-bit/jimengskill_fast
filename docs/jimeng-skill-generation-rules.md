## 转换原则

1. 先判断视频内容适合做「知识/规则型」「单工具封装型」还是「复合型」Skill。
2. 必须显式写出工具名和调用条件，不要只写“生成”“处理”。
3. 信息收集要用字段表或表单规范表达，区分必填与可选。
4. 必填信息缺失时应停止并追问；可选信息要给默认值。
5. 执行流程要区分用户参与阶段和自主执行阶段。
6. Prompt 规范要可执行，避免只写抽象审美词。
7. 要写清楚该 Skill 不处理什么，避免边界过宽。
8. 保留来源链接，但不要大段照搬博主原文；应转成方法论、参数模板和执行约束。

## 可直接喂给 GPT 的提示词

```text
你是即梦自定义 Skill 写作助手。请根据我提供的视频转写稿、原链接、截图备注和参考信息，生成一个符合即梦 Skill 风格的标准 Markdown。

输入资料：
1. 视频原链接：{{source_url}}
2. 平台/博主：{{platform_author}}
3. 视频转写稿：{{transcript}}
4. 截图/人工备注：{{notes}}
5. 该视频最值得复用的点：{{core_value}}

写作要求：
- 输出必须是完整 Markdown。
- 开头必须包含 YAML frontmatter：name 和 description。
- description 要说明本技能输出什么、支持哪些自定义参数、适用场景，并明确不处理什么。
- 正文必须包含这些一级/二级结构：
  1. # 技能名称
  2. ## 角色定位
  3. ## 工具使用规范
  4. ## 执行流程
  5. ## Prompt 设计规范
  6. ## 表单设计规范
  7. ## 全局约束
- 工具使用规范必须用表格，列为：工具名称 / 调用时机 / 说明。
- 执行流程必须分 Phase，写清楚目标、进入条件、执行步骤、输出产物、用户交互。
- 表单设计规范必须列出字段、是否必填、预设选项、是否允许自定义输入、缺失时处理。
- Prompt 设计规范必须写出提示词元素顺序，并提供一个可直接复制测试的示例 Prompt。
- 必须显式选择即梦工具名，例如 text2image、image2image、text2video、image2video、multi_modal2video、video_editor、generate_form_for_info_collection 等。
- 如果视频内容只是方法论或风格规则，可以写成知识/规则型 Skill，并说明无需直接调用生成工具。
- 不要照搬转写稿口播，提炼成可执行规则。
- 保留来源链接在文末「参考来源」中。
- 不要编造视频里没有的信息；缺失处用合理默认值或写明需表单收集。

请生成最终 Skill Markdown。
```
